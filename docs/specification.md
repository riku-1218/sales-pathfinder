# 自律型営業リサーチエージェント "Sales-Pathfinder" 開発仕様書

## 1. プロジェクト概要

**プロダクト名**: Sales-Pathfinder（セールス・パスファインダー）

**コンセプト**: 「商談前の30分のリサーチを、0秒にする」  
営業先企業の課題を特定し、自社商材の提案シナリオまで自動生成する最強の営業アシスタント。

### 解決する課題

1. インサイドセールスや営業担当が、商談前に相手企業のHP、中計、ニュースを調べるのに時間がかかりすぎている
2. 調査が浅いまま電話/訪問してしまい、「ウチのこと何も知らないね」と門前払いされる
3. 「相手の課題」と「自社商材」を結びつける提案トークを考えるのが苦手な若手が多い

**ターゲット**: B2B企業のインサイドセールス（IS）、フィールドセールス（FS）

---

## 2. システムアーキテクチャ & 技術スタック

| レイヤー | 採用技術 | 選定理由 |
|---------|---------|---------|
| Orchestration | Dify (Workflow Mode) | 複雑な思考プロセス（調査→課題抽出→提案作成）を構築するため |
| LLM (Brain) | Gemini 1.5 Flash | 長文の「中期経営計画」や「求人一覧」をまるごと読み込ませて分析するため（Long Context） |
| Search (Eyes) | Tavily API | ニュースやIR情報に特化した検索が可能 |
| Logic (Code) | Google Cloud Functions | 求人サイト等の複雑なHTMLからノイズを除去し、LLMが理解しやすい形に整形するため（Python） |
| Database | Dify Knowledge (RAG) | 「自社商材のカタログ・事例集」を登録し、提案の根拠にする |
| Integration | Make | 調査結果をCRM（顧客管理システム）やSlackに即時連携 |

## 3. 機能要件詳細

### 3-1. 入力インターフェース

- **必須**: `company_name` (例: 株式会社Target), `company_url`
- **オプション**: `sales_scenario` (例: 新規開拓 / 既存アップセル / 休眠掘り起こし)

### 3-2. 処理ロジック (Dify Workflow)

#### マルチアングル調査 (Scout Agent)

1. **経営視点**: 「中期経営計画」「社長メッセージ」を検索 → 「会社が今後どこに投資しようとしているか」を抽出
2. **現場視点**: 「採用情報（Wantedly等）」「社員口コミ（OpenWork等）」を検索 → 「今、現場で何の人材/スキルが不足しているか」を抽出
   - GCF処理: 取得した求人ページのHTMLから、ナビゲーションやフッター等のノイズを削除し、職種要件（必須スキル・歓迎スキル）のみを抽出
3. **直近の話題**: 「プレスリリース」「ニュース」を検索 → トークのきっかけ（アイスブレイク）を探す

#### 課題仮説の構築 (Hypothesis Agent)

収集した情報を統合し、「この会社は〇〇を目指しているが、××のリソースが不足している」という仮説を立てる

#### 提案ロジック生成 (Sales Strategist)

- **入力**: [相手の課題仮説] + [自社商材ナレッジ(RAG)]
- **思考**: 「相手の『××不足』という課題は、自社商材の『△△機能』で解決できる。なぜなら類似事例□□があるからだ。」
- **出力**:
  - フック: 「御社の中期経営計画にある『DX推進』について拝見しました...」
  - 課題指摘: 「特にエンジニア採用を強化されていますが、リソース不足がボトルネックではありませんか？」
  - 提案: 「弊社のツールを使えば、現状の人員で生産性を2倍にできます。」

### 3-3. 出力フォーマット

Google Docs等に以下の「商談準備シート」を出力:

```
【商談準備シート】株式会社〇〇

■ 企業概要サマリ: 売上、規模、事業内容（3行で）

■ キーマンの関心事（経営課題）:
  - 「2026年までに海外売上比率を50%にする」目標あり。

■ 現場の課題（推測）:
  - 英語ができる営業マネージャーを急募中 → 海外展開の人材が不足？

■ 【自社商材の導入余地（提案シナリオ）】:
  - 提案切り口: グローバル対応の営業支援ツールとして提案。
  - キラートーク: 「海外展開における現地スタッフ教育にお困りではありませんか？」
  - アイスブレイクネタ: 先週発表された〇〇社との提携ニュースについて。
```

---

## 4. 非機能要件

- **推論の根拠提示**: 「なぜ人材不足と思ったのか？」の根拠（例：求人ページのURL）を必ず明記する
- **スピード**: 商談直前でも使えるよう、実行から出力まで90秒以内を目指す

---

## 5. ロードマップ

- **v1.0**: 1社ずつの手動調査
- **v2.0**: リスト(CSV)を読み込ませ、100社分の「アプローチ優先度」と「トーク案」を一括生成する「リスト・スクリーニング機能」の実装
